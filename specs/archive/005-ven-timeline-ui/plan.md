# Implementation Plan: VEN Timeline UI

**Branch**: `005-ven-timeline-ui` | **Date**: 2026-03-16 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/005-ven-timeline-ui/spec.md`

## Summary

Implement the `controller/timeline.rs` backend module (currently a stub), expose `GET /timeline/{asset_id}` and `GET /timeline/all` HTTP endpoints, then rebuild the VEN controller-v2 UI to consume them. The data pipeline moves from frontend-assembled data (trace + plan + rates joined in `dataBuilders.ts`) to backend-assembled, pre-enriched timeline slices. Four frontend diagram bugs (battery/base_load missing, PV wrong data, unstable X-axis, `nowMs` not memoized) are resolved as a natural consequence. Extended per-cell time window toggles, schema-driven simulation controls, and the `useRates` ‚Üí `useTariffs` rename are also delivered.

## Tech Stack Decisions

### Backend: `build_asset_timeline` function

Pure function in `VEN/src/controller/timeline.rs`:

```rust
pub fn build_asset_timeline(
    asset_id: &str,
    history: &HashMap<String, AssetHistoryBuffer>,
    plan: &Plan,
    now: DateTime<Utc>,
    window: TimeWindow,
) -> Option<Vec<AssetTimelinePoint>>
```

- Returns `None` if `asset_id` not in `history` and not `"grid"` ‚Üí handler returns 404
- **Past**: calls `history[asset_id].to_timeline(Some((now - hours_back, now)))`
- **Future**: iterates `plan.firm_slots + plan.flexible_slots`; for each slot in `[now, now + hours_forward)`:
  - Finds allocation for `asset_id` ‚Üí `power_kw`
  - Computes `cost_rate_eur_h = power_kw * slot.import_price_eur_kwh`
  - Computes `co2_rate_g_h = power_kw * slot.co2_g_kwh`
  - Creates `AssetTimelinePoint { ts: slot.start, values: {...} }`
- Merges past + future, sorts by `ts`

**Grid special case** (`asset_id = "grid"`):
- Past: `history["grid"]` (net site power stored per-tick)
- Future: `PlanTimeSlot.net_import_kw / net_export_kw` + tariff fields

### Backend: HTTP handlers

```rust
// In main.rs ‚Äî registered BEFORE /:asset_id
async fn get_timeline_all(
    State(ctx): State<AppCtx>,
    Query(params): Query<TimelineParams>,
) -> impl IntoResponse

async fn get_timeline(
    State(ctx): State<AppCtx>,
    Path(asset_id): Path<String>,
    Query(params): Query<TimelineParams>,
) -> impl IntoResponse
```

`TimelineParams`: `hours_back: Option<f64>`, `hours_forward: Option<f64>` (defaults 1.0).

Lock order: acquire `sim`, then `state` (consistent with existing lock order in main.rs).

### Frontend: `AssetTimelineChart` data access

```typescript
// Before (flat named fields):
point.powerKw

// After (values map):
point.values["power_kw"] ?? 0
```

Chart `dataKey` uses recharts dot-notation: `"values.power_kw"` ‚Äî recharts can traverse nested objects.

### Frontend: Extended window toggle

```typescript
const [extended, setExtended] = useState(false);

const { hoursBack, hoursForward } = extended
  ? EXTENDED_WINDOWS[assetId]   // per-asset lookup table
  : { hoursBack: 1, hoursForward: 1 };

const { data: timeline } = useTimeline(assetId, hoursBack, hoursForward);
```

`EXTENDED_WINDOWS` constant:
```typescript
const EXTENDED_WINDOWS: Record<string, { hoursBack: number; hoursForward: number }> = {
  ev:      { hoursBack: 1,  hoursForward: 24 },
  battery: { hoursBack: 1,  hoursForward: 24 },
  grid:    { hoursBack: 0,  hoursForward: 24 },  // GridTariffCell uses "grid"
};
// Heater, PV, BaseLoad: no entry ‚Üí no toggle rendered
```

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Recharts `dataKey` nested object access (`"values.power_kw"`) may not work | Test early; fallback: map `values` to flat named fields only inside the chart component (not the hook) |
| Plan may be `None` at startup (no plan yet generated) | `build_asset_timeline` accepts `&Plan`; caller passes a default empty plan or guards with `Option<&Plan>`; future section is empty |
| Lock ordering: `sim` + `state` acquired in wrong order ‚Üí deadlock | Follow existing pattern in main.rs handlers (sim first, then state); verify in code review |
| `AssetHistoryBuffer["grid"]` may not exist (grid history not yet written by monitor.rs) | Check if speckit 004 wired grid history. If not, fall back to empty past section for grid; add grid history write as a task if missing |
| `DynamicControl` override merging with full-replace POST | Confirm `POST /sim/override` is full-replace (it is); client must send full override state, not just the changed field |
